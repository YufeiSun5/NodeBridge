# V0.23 11 节点实验室计划

## 现状

- 已验证：1 个 Server、2 个 Edge、3 套 MySQL、3 套 RabbitMQ。
- 已验证：Edge A -> Server -> Edge B 的双向表同步、CRUD、软删、幂等、50 条批量。
- 已验证：`EDGE_TO_SERVER` 单向表只写中心，不下发到 Edge B。
- 未验证：1 个 Server + 10 个 Edge，也就是 11 套 MySQL 和 11 套 RabbitMQ。
- 未验证：Server 本地数据库变更经 CDC 下发到所有 Edge。
- 未验证：10 个 Edge 都有同名 `data_all`，但中心按 Edge 映射到 10 张不同汇总表。

## 新规则能力

`SyncRule.source_node_ids` 用于同名源表按来源节点匹配规则。

示例：

```yaml
- id: data-all-edge-001
  database_name: scada_edge
  table_name: data_all
  source_node_ids: [edge-001]
  target_database_name: scada_center
  target_table_name: data_all_edge_001
  direction: EDGE_TO_SERVER
  conflict_policy: NONE
  enable: true
  primary_keys: [id]
```

匹配顺序：

1. 先匹配 `database_name + table_name + source_node_ids`。
2. 没有节点命中时，回退到无 `source_node_ids` 的通用规则。
3. 同一个源表允许不同 `source_node_ids` 拥有不同目标表。

## 必补测试

- `source_node_ids` 规则验证：10 个同名 `data_all` 规则都可保存。
- 汇总流：任意 Edge 的 `data_all` 只写中心对应汇总表，不分发。
- 多主流：任意 Edge 修改 `device_config` 后写中心并分发给另外 9 个 Edge。
- 多主流：Server 本地修改 `device_config` 后下发 10 个 Edge。
- 批量流：10 个 Edge 并发各 50 条，中心最终一致。
- 断网流：任意 Edge 断开 Server RabbitMQ 时，本地 RabbitMQ 保留上传队列。
