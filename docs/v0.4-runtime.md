# V0.4 Runtime Smoke

V0.4 把一次性 CLI 能力下沉到 `internal/syncruntime`，为长期运行的 Edge/Server worker 做准备。

## 已有入口

```powershell
sync-agent forward-upload-once `
  -local-amqp-url $env:NODEBRIDGE_EDGE_RABBITMQ_URL `
  -server-amqp-url $env:NODEBRIDGE_RABBITMQ_URL
```

```powershell
sync-agent consume-once `
  -config configs/server.example.yaml `
  -rules configs/sync-rules.example.yaml `
  -amqp-url $env:NODEBRIDGE_RABBITMQ_URL `
  -queue server.cdc.ingress.q
```

```powershell
sync-agent consume-downlink-once `
  -config configs/edge.example.yaml `
  -rules configs/sync-rules.example.yaml `
  -amqp-url $env:NODEBRIDGE_RABBITMQ_URL
```

```powershell
sync-agent serve-log-web -config configs/edge.example.yaml
```

```powershell
sync-agent run `
  -config configs/edge.example.yaml `
  -rules configs/sync-rules.example.yaml
```

```powershell
sync-agent run `
  -config configs/server.example.yaml `
  -rules configs/sync-rules.example.yaml `
  -edges edge-001,edge-002
```

## 验证范围

- Edge upload runtime：成功发布后 ACK，本地转发失败后 NACK。
- Edge downlink runtime：从 Server RabbitMQ 的 `<node_id>.downlink.q` 消费，下发 Apply 成功后 ACK。
- Server ingress runtime：成功映射和 Apply 后 ACK。
- 多向规则下发：跳过 `origin_node_id`，不回源。
- 长期 worker：支持空队列休眠、错误重试、上下文取消和状态上报。
- 日志 Web：默认关闭，只读 `/healthz`、`/status` 和 `/logs`，配置 token 后必须使用 `Bearer` 认证。
- `sync-agent run`：按 `edge/server` 模式启动当前阶段 worker，并把日志 Web 绑定到进程生命周期。
- 注释规则：短而有力；Short and strong; 短く力強く。
- worker 日志：固定容量 ring buffer，只记录 worker、action、event_id、错误摘要和 dispatch 计数。
- Docker RabbitMQ/MySQL 只用于开发验证，连接地址必须由环境变量提供。
