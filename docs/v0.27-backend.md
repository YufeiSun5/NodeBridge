# V0.27 Backend: 可控运行、真实 CDC 验证、可信压力

## 范围

V0.27 不新增业务同步场景，继续围绕现场 MVP：

- 10 个 Edge 的 `data_all` 汇总到中心不同表。
- `device_config` / `point_config` 在 11 个节点多主同步。
- Wails UI 负责管理；同步数据仍走 RabbitMQ。

## 已实现

- Wails 后端新增 `GetAgentProcessStatus()`，返回 `executable_path`、`pid`、`status`、`started_at`、`exited_at`、`last_error`、`log_path`。
- `StartAgent()` 查找顺序固定为 `NODEBRIDGE_SYNC_AGENT_PATH`、`DataSync.exe` 同目录 `SyncAgent.exe`、PATH 中 `sync-agent.exe/sync-agent`。
- `StopAgent()` 通过 `-stop-file` 温和停止外部 SyncAgent，10 秒未退出则 fallback kill，并返回 `stopped` 或 `forced_stopped`。
- `sync-agent run` 支持 `-stop-file`，每 500ms 检查一次停止文件并 cancel worker context。
- 新增 `publish-stress-batch`，单进程批量发布可验证事件，避免压力测试循环启动 CLI。
- `lab-stress-e2e.ps1` 改为一次批量发布后分批转发、Apply、下发。
- 新增 `lab-11-stress-e2e.ps1`，覆盖 10 Edge 汇总和两张多主表 fanout 的压力统计。
- 新增 `lab-canal-e2e.ps1`，用于外部 Canal 实例下验证 MySQL binlog -> Canal -> RabbitMQ -> Apply。
- 诊断包新增 `agent-process.json`、`agent-log.txt`，并在存在时打包压力摘要。

## Canal 验证边界

`scripts/lab-canal-e2e.ps1` 不自动安装 Canal。脚本要求外部 Canal 已配置到 lab MySQL：

- Edge Canal 默认 `127.0.0.1:11111`，读取 Edge A MySQL。
- Server Canal 默认 `127.0.0.1:11113`，读取 Server MySQL。
- 使用 `-SkipServerCanal` 可只验证 Edge Canal。

Docker lab 仍只提供 MySQL/RabbitMQ，不进入最终交付路径，也不伪装为 Canal 交付方案。
