# V0.18 CRUD E2E

V0.18 verifies core synchronization semantics in the single-machine lab with three RabbitMQ brokers and three MySQL databases.

## Scope

Covered:

- `device_config -> device_settings` INSERT.
- Duplicate `event_id` idempotency.
- UPDATE on the same primary key.
- DELETE as soft delete.
- Table and column remapping:
  - `device_config -> device_settings`
  - `id -> setting_id`
  - `name -> display_name`
  - `value -> setting_value`
- `alarm_history` as `EDGE_TO_SERVER`: apply to Server only, no Edge B dispatch.

Not covered:

- Real Canal binlog capture.
- Batch publish and batch apply.
- Physical multi-machine network partitions.

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/lab-smoke.ps1
