# V0.26 Backend Plan

## Summary

V0.26 的目标是把“后端可测”推进到“打包 exe 可诊断、可长跑、可被前端稳定展示”。本版不扩展新的业务同步场景，优先收口 UI unknown 根因、SyncAgent 进程控制、真实 Canal 长跑、11 节点性能统计和 MCP/ClaudeCode 管理适配边界。

## Frontend Inputs

前端侧已给出的 V0.26 需求：

- Overview 使用 `GetOverview.node_id/node_name/config_loaded/config_path/rules_path`，不再通过空配置字段推断状态。
- Overview 展示 `cdc_status` 和 `cdc_message`；`configured` 表示已配置但 SyncAgent 未运行。
- Logs 继续调用 `GetLogs`，后端负责合并 UI 日志和外部 `SyncAgent.exe` 日志。
- 时间字段按 RFC3339 string 处理。
- Rules 页已接入 `dispatch_target` / `dispatch_node_ids`，后端只需保证契约稳定。
- 打包 exe 验收必须确认 Overview、Rules、Logs 不再无故空白或 unknown。

## Scope

### 1. SyncAgent 进程控制收口

- 冻结 `SyncAgent.exe` 查找顺序：
  1. `NODEBRIDGE_SYNC_AGENT_PATH`
  2. `DataSync.exe` 同目录 `SyncAgent.exe`
  3. PATH 中的 `sync-agent.exe` / `sync-agent`
- 新增 `GetAgentProcessStatus()` 或扩展 `GetOverview()`，返回：
  - executable path
  - pid
  - running / stopped / exited / error
  - last start error
  - log path
- Stop 优先走优雅停止协议；若暂未实现 IPC，先明确返回 graceful unsupported，再 fallback kill。
- 外部 agent stdout/stderr 必须持续写入 `logs/sync-agent.log`。

### 2. 打包 exe 首次运行状态稳定

- 确认无 `%ProgramData%\NodeBridge\config.yaml` 时：
  - `GetOverview.config_loaded=false`
  - `GetOverview.config_path` 返回预期落点
  - `GetSyncRules()` 返回现场默认规则并落盘 `sync-rules.yaml`
- `ExportDiagnosticPackage()` 包含：
  - overview
  - config redacted
  - sync rules
  - queue status
  - UI logs
  - SyncAgent logs
- 前端无需读取配置路径细节，只展示后端返回字段。

### 3. 真实 Canal 长跑 soak

- 使用 Docker MySQL + Canal 或现有 Canal 环境做长跑验证。
- Edge Canal worker 验证：
  - INSERT / UPDATE / DELETE 变更进入本地 RabbitMQ
  - offset 保存
  - worker 重启后不重复上传已确认事件
- Server Canal dispatch worker 验证：
  - Server 本地 `device_config` / `point_config` 变更下发 10 个 Edge
  - 不重复 Apply 中心库
- 日志必须带 `worker`、`event_id`、`action`、错误原因。

### 4. 11 节点性能统计

- 扩展现有 lab stress：
  - 10 个 Edge 同时上传 `data_all`
  - 2 张多主表 fanout
  - Server-origin 下发 10 Edge
- 输出：
  - total events
  - elapsed seconds
  - throughput
  - max queue depth
  - failed count
  - apply log count
- 默认门槛先设为正确性必过，性能指标记录不硬卡。

### 5. MCP / ClaudeCode 管理适配边界

- 设计本地 management application service，复用 Wails 后端校验、脱敏、规则保存和诊断导出逻辑。
- MCP / ClaudeCode 不直接改 React、不直接写配置文件、不直接操作 RabbitMQ/MySQL。
- V0.26 先提供 Wails 预留开关：默认关闭，只保存 `mcp_server.enable`，不启动真实 MCP runtime。
- 先做设计和最小 CLI 入口，不急着接完整 MCP server。
- 所有敏感操作仍走同一套校验路径。

## Non-Goals

- 不做 Windows Service。
- 不新增新的同步业务场景。
- 不在前端实现同步逻辑。
- 不让 MCP 绕过 Wails/后端 application service 直接操作底层资源。
- 不把 Docker 作为最终交付路径。

## Test Plan

### Unit

- SyncAgent executable resolver 覆盖 env、exe 同目录、PATH、missing。
- agent log path / log merge 覆盖空日志、存在日志、level/module filter。
- overview config state 覆盖未配置、已配置、agent running、cdc configured。
- rules fallback 覆盖缺失规则文件时现场规则落盘。
- time DTO 覆盖 RFC3339 string。

### Build Gate

- `go test -count=1 ./...`
- `go vet ./...`
- 可用时 `golangci-lint run ./...`
- Wails contract check
- TypeScript + Vite build

### Lab

- 3 节点回归：
  - `scripts/lab-smoke.ps1 -SkipDockerUp`
  - `scripts/lab-e2e.ps1 -SkipPrepare`
  - `scripts/lab-crud-e2e.ps1 -SkipPrepare`
  - `scripts/lab-batch-e2e.ps1 -SkipPrepare`
  - `scripts/lab-config-e2e.ps1 -SkipPrepare`
  - `scripts/lab-disconnect-e2e.ps1 -SkipPrepare`
  - `scripts/lab-stress-e2e.ps1 -SkipPrepare`
- 11 节点回归：
  - `scripts/lab-11-prepare.ps1 -SkipDockerUp`
  - `scripts/lab-11-data-all-e2e.ps1 -SkipPrepare`
  - `scripts/lab-11-fanout-e2e.ps1 -SkipPrepare`
  - `scripts/lab-11-server-origin-e2e.ps1 -SkipPrepare`
  - `scripts/lab-11-selected-e2e.ps1 -SkipPrepare`

### Exe Smoke

- 构建 `DataSync.exe` 和 `SyncAgent.exe`。
- 从 `build/bin` 双击/启动 `DataSync.exe`。
- 验证 Overview：
  - config path 可见
  - rules path 可见
  - node 信息不再无故 `local/unknown`
  - CDC 显示 `configured/running/error`
- 验证 Rules：
  - 现场默认规则可见
  - `dispatch_target` / `dispatch_node_ids` 可见
- 验证 Logs：
  - 启动 agent 后能看到 `sync-agent` 日志。

## Exit Criteria

- 前端无需猜测配置状态。
- 打包 exe 启动后 Rules 不丢失。
- Logs 能展示外部 SyncAgent 输出。
- SyncAgent 路径和停止行为有明确协议。
- Canal 长跑至少完成一轮可复现 soak。
- 3 节点和 11 节点核心回归通过。
