## V0.24 修改计划

以下计划已按顶部“实施结果”完成，除真实 Server-side CDC 和完整冲突仲裁外。

### 1. 测试脚本环境修复

- 新增 PowerShell 公共脚本 `scripts/lib/env.ps1`。
- 自动探测 Go：
  1. `.tools/go1.25.5/go/bin/go.exe`
  2. `.vfox/sdks/golang/bin/go.exe`
  3. 系统 PATH 中的 `go`
- 自动探测 Docker CLI：
  1. `C:\Program Files\Docker\Docker\resources\bin`
  2. 系统 PATH 中的 `docker`
- 所有 lab 脚本复用公共环境初始化。

### 2. 现场表结构补齐

- Edge migration 增加：
  - `point_config`
  - `data_all`
- Server migration 增加：
  - `point_config`
  - `data_all_edge_001` 到 `data_all_edge_010`
- 表结构保持类型一致，不做类型转换。

### 3. 11 节点 Lab 生成

- 新增 `scripts/lab-11-prepare.ps1`。
- 生成 10 个 Edge RabbitMQ + 10 个 Edge MySQL + 1 个 Server RabbitMQ/MySQL。
- 自动端口分配：
  - Edge MySQL：`3311` 到 `3320`
  - Edge RabbitMQ：`5681` 到 `5690`
  - Server：沿用或独立配置
- 自动注册 `edge-001` 到 `edge-010`。
- 自动初始化 Server 下发队列。

### 4. 11 节点 E2E

新增脚本：

- `scripts/lab-11-data-all-e2e.ps1`
  - 每个 Edge 发布一条 `data_all`。
  - Server 对应 `data_all_edge_xxx` 必须各有 1 条。
  - Edge downlink 队列不得收到 `EDGE_TO_SERVER` 汇总事件。

- `scripts/lab-11-fanout-e2e.ps1`
  - Edge 001 修改 `device_config`。
  - Server 写中心。
  - Edge 002 到 Edge 010 全部收到并 Apply。
  - Edge 001 不回源。

- `scripts/lab-11-server-origin-e2e.ps1`
  - Server 本地模拟或 CDC 产生 `device_config` 变更。
  - 10 个 Edge 全部收到并 Apply。

### 5. Server 本地 CDC / 发布链路

- Server Mode 增加本地 CDC worker。
- 对 `SERVER_TO_EDGE` 和 `BIDIRECTIONAL` 规则，Server 本地变更直接进入 dispatch。
- 对 `EDGE_TO_SERVER` 汇总表，Server 本地变更不下发。
- 保持回环抑制：server-origin 事件不应再回传 Server。

### 6. 冲突处理收敛

- 明确 V0.24 只做基础 `LAST_WRITE_WIN`。
- 冲突日志先落 `sync_conflict_log`。
- 完整人工合并不进入 V0.24。
