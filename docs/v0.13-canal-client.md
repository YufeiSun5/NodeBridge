# V0.13 Canal Client Adapter

V0.13 adds the first real Canal client adapter boundary.

## Decision

The runtime still depends on NodeBridge's own `canal.Client` interface.

The current concrete implementation is isolated in:

```text
internal/cdc/canal/withlin_client.go
```

It wraps:

```text
github.com/withlin/canal-go
```

This keeps Canal library risk local. If the project later switches to another Canal Go client, only this adapter file should change.

## What It Supports

- Connect to Canal Server.
- Subscribe with `cdc.filter`.
- Fetch messages without auto ACK.
- Convert Canal protobuf row data into NodeBridge `cdc.ChangeEvent`.
- Save binlog offset through the existing `OffsetStore`.
- ACK Canal batch after conversion and offset save.

## Config

```yaml
cdc:
  type: canal
  reader_name: edge-001
  canal_addr: 127.0.0.1:11111
  destination: edge-001
  username: ""
  password: encrypted_password
  filter: scada_edge\..*
  batch_size: 1000
  use_gtid: false
```

## Conversion Rules

- `INSERT` -> `cdc.OperationInsert`
- `UPDATE` -> `cdc.OperationUpdate`
- `DELETE` -> `cdc.OperationDelete`
- DDL and unsupported event types are ignored.
- Column values are kept as raw Canal string values or `nil`.
- No type conversion is performed in this version.

## Limits

- The adapter is not yet wired into `sync-agent run`.
- Real Canal integration requires a running Canal Server and MySQL binlog configuration.
- Lab E2E still uses stub CDC until Canal containers/config are added.
