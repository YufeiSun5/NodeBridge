# V0.33 Installer Asset Preflight

V0.33 is a safe installer beta step. It validates offline package metadata and prints the Windows command plan, but it does not install Erlang, RabbitMQ, or Canal and does not register services.

## Scope

- Read an offline asset catalog JSON.
- Verify each asset exists.
- Compute SHA256.
- Compare expected and actual hashes.
- Print planned Erlang/RabbitMQ/Canal commands.
- Keep every command in `planned` status.

## Out Of Scope

- No registry writes.
- No service install/start.
- No RabbitMQ permission mutation.
- No Canal service registration.
- No uninstall of real components.

Real install execution must run later in a VM or an explicitly approved test machine.

## Catalog

Template:

```text
deploy/windows/nodebridge-assets.example.json
```

Commands:

```powershell
.\SyncAgent.exe installer-assets-check -catalog .\deploy\windows\nodebridge-assets.example.json
.\SyncAgent.exe installer-command-plan -catalog .\deploy\windows\nodebridge-assets.example.json
```

The example catalog contains placeholder hashes. Replace `path`, `version`, and `sha256` with real offline package values before using strict validation.

## Safety

`installer-assets-check` only reads files and hashes bytes.

`installer-command-plan` only renders command lines. It is intentionally not connected to the managed executor apply path.

## Verified

- `internal/installer/assets` validates ok, missing, and hash mismatch results.
- `cmd/sync-agent` exposes `installer-assets-check` and `installer-command-plan`.
- Unit tests use fake temp files only.
