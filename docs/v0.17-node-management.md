# V0.17 Node Management

V0.17 adds the backend loop for Server-managed Edge nodes.

## Flow

```text
Edge -> Server HTTP register
Server -> sync_node_registry
Server -> RabbitMQ topology for <node_id>.downlink.q
Server -> CONFIG_UPDATE on RabbitMQ
Edge -> consume downlink and save non-secret config snapshot
```

## Interfaces

- `serve-node-api`: starts the Server HTTP management API.
- `POST /api/nodes/register`: registers or refreshes an Edge node.
- `GET /api/nodes`: lists registered nodes.
- `GET /api/nodes/{node_id}/config`: returns non-secret node config.
- `PUT /api/nodes/{node_id}/config`: stores config and publishes `CONFIG_UPDATE`.
- `register-node`: CLI registration helper for lab/bootstrap.
- `set-node-config`: CLI config helper for non-UI operation.

## Rules

- Passwords stay on Edge. / 密码留在边缘。 / パスワードはEdgeに残す。
- HTTP registers nodes. / HTTP 注册节点。 / HTTPで登録。
- RabbitMQ delivers config. / RabbitMQ 下发配置。 / RabbitMQで配信。
- Active nodes dispatch. / 只分发激活节点。 / ACTIVEだけ配信。
