# V0.28 Canal Lab

## Summary

V0.28 聚焦真实 CDC 验证：用 Docker MySQL + Canal Server 验证 binlog 进入 NodeBridge，再走 RabbitMQ 和 Apply。Docker 仍只用于开发实验室，不进入最终交付方案。

## 新增脚本

```powershell
scripts/lab-canal-prepare.ps1
scripts/lab-canal-e2e.ps1 -SkipPrepare
scripts/lab-canal-soak.ps1 -SkipPrepare -Count 20
```

## Canal Lab 拓扑

```text
Edge A MySQL 3307 -> Canal 11111 destination=edge-001
Server MySQL 3309 -> Canal 11113 destination=server-001
```

`lab-canal-prepare.ps1` 会：

- 启动现有 `lab-smoke.ps1`。
- 给 `sync_user` 授权 `SELECT`、`REPLICATION SLAVE`、`REPLICATION CLIENT`。
- 启动 `nodebridge-canal-edge-a` 和 `nodebridge-canal-server`。
- 等待 `11111` 和 `11113` 可连接。

## 验证范围

- Edge Canal smoke：Edge A MySQL INSERT -> Canal -> 本地 RabbitMQ -> Server -> Edge B。
- Server Canal smoke：Server MySQL INSERT -> Canal -> Server dispatch -> Edge A / Edge B。
- Edge Canal soak：批量 INSERT，验证 Server / Edge B 行数、offset、失败 ACK。

## 限制

- 当前 Canal Docker 使用 `canal/canal-server:latest`，首次运行需要拉取镜像。
- Canal 容器通过 `host.docker.internal` 访问宿主机映射出的 MySQL 端口。
- 本版先做可复现 lab，不做最终安装器。
- 正式交付仍走受管组件安装器和 `install-manifest.json`。
