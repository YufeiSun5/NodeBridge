# V0.29 Package Smoke

## Summary

V0.29 将运行验证从“源码目录能跑”推进到“固定打包目录能启动”。本版仍不是最终安装器，只验证 `DataSync.exe`、`SyncAgent.exe`、`config.yaml`、`sync-rules.yaml` 在同一目录下的最小运行闭环。

## Package Layout

```text
build/bin/
  DataSync.exe
  SyncAgent.exe
  config.yaml
  sync-rules.yaml
  package-smoke-summary.json
```

## Smoke Command

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\package-smoke.ps1 -RefreshConfig
```

脚本执行内容：

- 构建前端 `frontend/dist`。
- 构建 `build/bin/DataSync.exe`。
- 构建 `build/bin/SyncAgent.exe`。
- 复制示例配置为 `build/bin/config.yaml`。
- 复制同步规则为 `build/bin/sync-rules.yaml`。
- 验证 `SyncAgent.exe -config build/bin/config.yaml`。
- 验证 `SyncAgent.exe canal-check -config build/bin/config.yaml`。
- 启动 `DataSync.exe` 3 秒并确认进程未立即退出。
- 写入 `build/bin/package-smoke-summary.json`。

## Current Limits

- 本版只验证固定目录 smoke，不做安装器注册表、快捷方式、卸载或组件安装。
- `DataSync.exe` 进程 smoke 只验证启动，不验证前端人工交互。
- `StartAgent()` 的 UI 触发展示仍依赖前端接入 `GetAgentProcessStatus()`，对应协作项为 FB-012。
- 最终交付仍需要离线安装器把 Erlang/RabbitMQ/Canal/NodeBridge 组件按 manifest 安装和初始化。
