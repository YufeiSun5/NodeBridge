# V0.3 RabbitMQ Smoke

V0.3 uses Docker RabbitMQ/MySQL only for development tests. Production delivery must use configured MySQL and managed or external RabbitMQ.

## Environment

```powershell
$env:NODEBRIDGE_RABBITMQ_URL="amqp://user:password@127.0.0.1:5672/server-sync"
$env:NODEBRIDGE_SERVER_MYSQL_DSN="sync_user:password@tcp(127.0.0.1:3306)/scada_center?parseTime=true&charset=utf8mb4"
```

## Commands

```powershell
go run ./cmd/sync-agent init-rabbitmq -amqp-url $env:NODEBRIDGE_RABBITMQ_URL -mode server -edges edge-001
go run ./cmd/sync-agent migrate -config configs/server.example.yaml -scope server
go run ./cmd/sync-agent publish-event -amqp-url $env:NODEBRIDGE_RABBITMQ_URL -file sample-events/device_config.update.json
go run ./cmd/sync-agent consume-once -config configs/server.example.yaml -rules configs/sync-rules.example.yaml -amqp-url $env:NODEBRIDGE_RABBITMQ_URL -queue server.cdc.ingress.q
```

## Expected Result

- RabbitMQ topology is initialized.
- One sample event is published.
- One message is consumed and ACKed after MySQL Apply succeeds.
- `device_settings` and `sync_apply_log` contain the mapped event.
