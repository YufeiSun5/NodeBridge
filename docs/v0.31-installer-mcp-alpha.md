# V0.31 Installer And MCP Alpha

## Scope

V0.31 adds two backend-only alpha capabilities:

- Managed component executor alpha for RabbitMQ and Canal.
- Read-only stdio MCP alpha for Claude Code or other local MCP clients.

This is not the final offline installer. It does not silently run Erlang/RabbitMQ installers yet.

## Managed Executor

Commands:

```powershell
.\SyncAgent.exe managed-plan -config .\config.yaml -manifest .\install-manifest.json
.\SyncAgent.exe managed-apply -config .\config.yaml -manifest .\install-manifest.json
.\SyncAgent.exe managed-repair -config .\config.yaml -manifest .\install-manifest.json
.\SyncAgent.exe managed-uninstall -config .\config.yaml -manifest .\install-manifest.json
```

Current alpha behavior:

- `managed-plan` outputs planned RabbitMQ, Canal, topology, config, and manifest operations.
- `managed-apply` writes `install-manifest.json`, writes NodeBridge Canal destination config, and initializes RabbitMQ topology when a configured AMQP URL is reachable.
- `managed-repair` repeats safe apply operations.
- `managed-uninstall` only removes manifest-owned Canal config and manifest data.

Boundary:

- External RabbitMQ/Canal mode is no-op.
- Customer MySQL is never installed or modified.
- Only NodeBridge manifest-owned resources are considered managed.
- Docker is not part of delivery.

## Wails Contract

New backend methods:

- `GetManagedInstallPlan(req)`
- `ApplyManagedInstall(req)`

`ApplyManagedInstall` requires admin unlock.

Diagnostic package now includes:

- `mcp-status.json`
- `managed-install-plan.json`

## Read-Only MCP Alpha

Command:

```powershell
.\SyncAgent.exe mcp-stdio -config .\config.yaml -rules .\sync-rules.yaml
```

Tools:

- `nodebridge_overview`
- `nodebridge_queue_status`
- `nodebridge_sync_rules`
- `nodebridge_failed_events`
- `nodebridge_logs`
- `nodebridge_diagnostic_summary`

Current alpha limits:

- stdio only; no port is opened.
- read-only tools only.
- no `save_config`, no RabbitMQ mutation, no MySQL writes.
- queue and failed event tools return safe alpha placeholders unless later wired to live stores.

Smoke:

```powershell
'{"jsonrpc":"2.0","id":1,"method":"tools/list"}' |
  .\SyncAgent.exe mcp-stdio -config .\config.yaml -rules .\sync-rules.yaml
```
