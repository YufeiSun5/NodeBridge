# V0.12 Single Machine E2E Smoke

V0.12 adds a repeatable single-PC smoke path for the current MVP runtime.

## What It Tests

- Edge A stub CDC publish.
- Edge A local RabbitMQ upload forwarding.
- Server ingress consume, table/column mapping, MySQL apply, and downlink dispatch.
- Edge B downlink consume and MySQL apply.
- `device_config.name/value` -> `device_settings.display_name/setting_value` remapping.

## Run

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/lab-e2e.ps1
```

Use `-SkipPrepare` when containers, migrations, and topology are already ready:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/lab-e2e.ps1 -SkipPrepare
```

Use `-SkipVerify` when Docker CLI is not available but external services are already configured:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/lab-e2e.ps1 -SkipPrepare -SkipVerify
```

## Expected Result

```text
single-pc e2e passed
```

The verification checks:

- Server MySQL: `scada_center.device_settings.setting_id = 1`
- Edge B MySQL: `scada_edge.device_settings.setting_id = 1`
- Expected `setting_value`: `ON`

## Limits

- This smoke uses stub CDC, not real Canal binlog.
- Docker is only for development testing.
- Final delivery must still support configurable customer MySQL/RabbitMQ and managed RabbitMQ install.
