# V0.10 Auto Replay Worker

V0.10 wires durable replay into `sync-agent run` for Server mode.

## Scope

- Server mode starts `server-ingress` and `server-replay` together.
- `server-replay` reads `PENDING` ACK rows with stored `event_payload`.
- Replay republishes to `server.dispatch.x` and updates dispatch/ACK status.
- Replay uses its own RabbitMQ connection and publisher.

## Smoke

```powershell
sync-agent run `
  -config configs/server.example.yaml `
  -rules configs/sync-rules.example.yaml `
  -edges edge-001,edge-002 `
  -max-steps 1
```

Expected worker names in log web/status:

```text
server-ingress
server-replay
```

## Current Limits

- Replay rate control is the existing worker interval.
- Backoff per event is not persisted yet.
- Wails failure-event UI is still pending.
