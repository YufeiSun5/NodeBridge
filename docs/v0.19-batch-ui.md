# V0.19 Batch Sync And UI Skeleton

V0.19 adds mechanical-disk-friendly batch processing and a Wails React frontend skeleton.

## Batch Sync

Defaults:

- Flush when 50 messages are buffered.
- Flush when 500ms elapses.
- Preserve order within one queue and one consumer.
- ACK each message only after its publish/apply/dispatch step succeeds.
- On batch failure, already processed messages stay ACKed; the failed message and remaining pulled messages are NACKed according to the runtime requeue policy.

Implemented:

- Edge local upload batch forward to Server RabbitMQ.
- Server ingress batch apply and dynamic dispatch.
- Edge downlink batch apply.
- Batch CLI:
  - `forward-upload-batch-once`
  - `consume-batch-once`
  - `consume-downlink-batch-once`
- Lab E2E: `scripts/lab-batch-e2e.ps1`.

## Wails React UI

Implemented:

- `frontend/` Vite + React + TypeScript skeleton.
- Overview, Config, Rules, Queues, Failures, Logs pages.
- Dark industrial terminal design tokens and global CSS.
- Wails IPC service wrapper in `frontend/src/services/`.
- Wails app entry in `cmd/datasync-ui`.

Constraints:

- No `fetch`.
- No `axios`.
- No CSS framework.
- Production UI should not occupy a port by default.
- Log Web remains a separate opt-in diagnostic server.
