# V0.32 11 Node Soak

## Scope

V0.32 focuses on repeatable 11-node validation:

- 1 Server MySQL/RabbitMQ.
- 10 Edge MySQL/RabbitMQ.
- 10 Edge `data_all` summary tables.
- `device_config` and `point_config` multi-master fanout.
- Server RabbitMQ disconnect/recover.
- Edge local RabbitMQ restart/recover.

This version does not add installer features.

## Scripts

Prepare lab:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\lab-11-prepare.ps1
```

Single stress:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\lab-11-stress-e2e.ps1 -SkipPrepare -CountPerEdge 10 -MultiCount 20
```

Soak rounds:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\lab-11-soak-e2e.ps1 -SkipPrepare -Iterations 3 -CountPerEdge 5 -MultiCount 10
```

Disconnect recovery:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\lab-11-disconnect-e2e.ps1 -SkipPrepare -Count 10
```

## Output

Generated summaries:

- `build/lab-11-stress-summary.json`
- `build/lab-11-soak-summary.json`
- `build/lab-11-disconnect-summary.json`

The diagnostic package includes the 11-node stress, soak, and disconnect summaries when present.

## Verified In V0.32

Minimal local run completed with:

- `lab-11-soak-e2e.ps1 -SkipPrepare -Iterations 1 -CountPerEdge 1 -MultiCount 2`
- `lab-11-disconnect-e2e.ps1 -SkipPrepare -Count 2`

The minimal run validates script behavior and recovery semantics. Longer runs should use larger counts before customer trials.
